# Creative Ideation Filter

Use this reference for creative directions, campaign concepts, slogans/themes, content topics, social ideas, proposal creative, brand activations, or any "出创意" task.

The goal is not to produce more ideas. The goal is to filter out generic ideas that could be used by any competitor.

## Minimum Inputs

Before ideation, look for:

- **Product**: one-sentence product/category/business description.
- **True details**: at least 2-3 concrete facts, features, constraints, proof points, usage details, brand assets, or service mechanisms.
- **Audience scene**: one specific moment where the target user has a need, hesitation, friction, desire, or decision.
- **Output use**: campaign direction, social topic, slogan, launch concept, event idea, video script seed, proposal page, or other format.

If the product and scene are missing, ask for them. If the user wants to continue without them, state the assumption and mark the ideas as provisional.

## Required Process

1. Generate **8-10 raw directions** first. Keep them short; do not expand into full plans yet.
2. Run the competitor substitution test on every raw direction:
   - Ask: "Would this still work if I put a competitor's name on it?"
   - If yes, delete or rewrite it.
3. Run the fact/scene binding test:
   - Each surviving idea must name the product fact or audience scene it depends on.
   - If it cannot name one, delete it.
4. Keep **3 final directions** only, unless the user asks for a larger pool.
5. For each final direction, write one plain sentence explaining what the user does, sees, gets, or decides. Do not stop at an abstract concept name.
6. Tell the user which raw directions were eliminated and why only when the user asks for ideation, screening, or auditability. For 口径修改 or final-copy tasks, run the filtering internally and output the strongest version.

## Banned Creative Words

Avoid these unless quoting source material:

- 沉浸式、拉满、引爆、破圈、情绪价值、极致、颠覆、赋能、打造、全新升级
- Also avoid common empty proposal words when they act as decoration: 生态、闭环、矩阵、深度、多维度、全面提升、强势出圈.

Replace them with product facts, user actions, sensory details, proof objects, or concrete choices.

## Output Modes

### Clean Final Mode

Use this for "调整口径", "改成客户能看的说法", "帮我润一下", "给我一版最合适的", or final-copy tasks.

```text
[final copy]
```

If a risky assumption or internal wording issue was resolved, add at most 1-2 short notes after the copy.

### Ideation Review Mode

Use this when the user asks for creative directions, options, or wants to see the screening process.

For ideation review mode:

```text
原始方向池：
1. [direction] - [fact/scene it tries to use]
...

筛选后保留：
1. [direction name]
   大白话：用户会[do/see/get/decide].
   扣住的事实/场景：[specific fact or audience scene]
   为什么不是竞品通用：[reason]

淘汰说明：
- [eliminated direction]: [reason]
```

For client-ready ideation output, put the three final directions first and move eliminated directions into a short `【筛选说明】`. Do not include the raw pool in final client copy unless requested.

## Good Direction Criteria

A strong creative direction should pass at least three of these:

- It depends on a real product fact, not a category promise.
- It uses a concrete audience scene, not a broad demographic label.
- It creates a visible user action, object, comparison, ritual, choice, or story.
- It has a natural execution form: poster, short video, social topic, event, package, demo, landing page, community task, or sales script.
- It creates a reason for this brand/product to say it, not just any brand in the category.

## Weak Direction Patterns

Delete or rewrite ideas that:

- are only emotional labels: "陪伴感", "松弛感", "年轻态", "高级感".
- only rename the category benefit: "更高效的办公体验", "更健康的生活方式".
- rely on a vague atmosphere but no product behavior.
- require fake user stories, fake data, or unproven claims.
- need a big budget, celebrity, or channel trick to look interesting.
- sound like a title but do not imply an execution.

## Example Conversion

Weak:

```text
打造沉浸式健康生活体验。
```

Better:

```text
方向：把每天最容易放弃的那一步做成可见进度。
大白话：用户打开产品时，不是先看一堆健康理念，而是看到今天只差哪一步就能完成目标。
扣住的事实/场景：产品如果有连续记录、提醒、进度展示或低门槛任务机制，才能成立。
为什么不是竞品通用：它依赖产品能把复杂目标拆成每日可完成动作；没有这个机制的产品说不了。
```
